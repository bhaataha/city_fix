# 🔒 RULE #9: RESEARCH & PLAN BEFORE DEVELOPMENT

> **CRITICAL**: Before writing ANY code for a new feature or significant change, complete the full Research & Plan pipeline via `mcp_autopilot_planning`.

## Mandatory Pipeline:
1. `planning` (research) → Document goal + findings
2. `planning` (plan) → Implementation plan + tech decisions
3. `planning` (generate_workflow) → Auto-generate workflow
4. `planning` (generate_tasks) → Auto-generate TODO tasks
5. Escalate only if the plan changes scope, risk, or architecture materially
6. `planning` (approve) → Mark plan ACTIVE when manual approval is actually required
7. Start development on generated tasks

## Applies to: FEATURE, REFACTORING, INTEGRATION, UI (>3 files)
## Skip for: BUGFIX (<2 files), DOCUMENTATION, TESTING

**ENFORCEMENT: ALWAYS use planning pipeline for significant work. Zero exceptions.**
